

Project Proposal: HOME FUR GOOD

-----------------------------------------------------------------------------------
-----------------------------------------------------------------------------------

DESCRIPTION:

Home Fur Good is a dog adoption web application built using the Petfinder API. The site aims to help dogs—especially those in high-kill shelters—find permanent, loving homes. The homepage will feature a rotating spotlight section to highlight newly adoptable dogs or those with the most urgent needs. Users will be able to search for dogs based on detailed criteria including breed, age, weight, color, size, temperament, and more. Users can create accounts to save dogs they like and submit adoption applications. There will also be an admin login for shelter staff to view applications or feature certain dogs. The site will be fully responsive and accessible on both desktop and mobile devices.

-----------------------------------------------------------------------------------
-----------------------------------------------------------------------------------

STACK:

--Backend: Python, Flask, SQLAlchemy, PostgreSQL

--Frontend: HTML, CSS, JavaScript, Jinja templates

--API: Petfinder API (external), and a custom internal API for user data and saved dogs

--Optional Tools: WTForms, Flask-Login, Flask-WTF for forms and auth, Flask-Mail (stretch goal)

-----------------------------------------------------------------------------------
-----------------------------------------------------------------------------------

FOCUS:

This will be a full-stack application with a strong emphasis on both frontend interactivity (search filters, user dashboard) and backend logic (authentication, data storage, API interaction).

-----------------------------------------------------------------------------------
-----------------------------------------------------------------------------------

TYPE:
Web application, fully responsive for desktop and mobile users.

-----------------------------------------------------------------------------------
-----------------------------------------------------------------------------------

GOAL:

The goal is to create a compassionate and efficient tool for connecting potential adopters with adoptable dogs, focusing on those in high-risk situations. The platform will allow users to easily search for, learn about, and apply to adopt dogs. It will also give shelters a way to highlight dogs that need urgent attention.

-----------------------------------------------------------------------------------
-----------------------------------------------------------------------------------

USERS:

--Primary Users: Individuals looking to adopt a dog
--Secondary Users: Shelter administrators/staff managing dog visibility or viewing applications


-----------------------------------------------------------------------------------
-----------------------------------------------------------------------------------

DATA:

The app will pull data from the Petfinder API, focusing only on dogs. Data will include:
--Dog details (name, age, breed, location, shelter, photo, description, temperament, etc.)
--Flags or tags indicating high-kill shelter status or urgent adoption needs (if available)
--User-generated data such as saved dogs, applications, and profile preferences will be stored in a PostgreSQL database
--An internal API will be created to manage user accounts, saved dogs, and applications

-----------------------------------------------------------------------------------
-----------------------------------------------------------------------------------

APPROACH & PLANNING:

Database Schema (initial concept):
--Users (id, username, email, password_hash)
--SavedDogs (id, user_id, dog_id_from_API, saved_date)
--Applications (id, user_id, dog_id_from_API, application_text, submitted_at)
--Admins (id, username, password_hash)


Functionality Overview:
--Spotlight section featuring randomly selected or urgent dogs on the homepage
--Dog search with multiple filters
--User registration and login (secure with hashed passwords)
--User dashboard with saved dogs and submitted applications
--Admin login with access to view submitted applications
--Responsive layout for desktop and mobile
--Stretch: Email sign-up for weekly dog updates (Flask-Mail)


Challenges/Considerations:
--The Petfinder API may not explicitly flag dogs in high-kill shelters, so this may require creative use of location data or tags
--Rate limits or inconsistencies in Petfinder data may affect API reliability
--Authentication and user data must be securely handled
--Ensuring accessibility and good UX for both mobile and desktop users


Stretch Goals:
--Weekly digest emails featuring dogs in need
--Calendar integration for scheduling adoption appointments or meet-and-greets
--Filtering results to prioritize dogs in high-kill shelters (if feasible)
--Social sharing buttons for dog profiles
